# 초코창고(Choco Inventory) PWA 앱 검토 보고서

**작성일:** 2026년 5월 25일
**검토자:** Antigravity (Review Agent)

본 보고서는 실무 환경(Production-adjacent)에서의 사용성을 바탕으로, 현재 `choco-inventory` 앱의 코드 및 UI/UX를 분석한 결과입니다.

---

## 1. 모바일 일일 재고 실사 워크플로우 분석
현재 앱은 모바일 환경에 매우 최적화되어 있으며, 현장에서의 실사 속도를 높이기 위한 장치들이 잘 마련되어 있습니다.

- **강점:**
    - **커스텀 키패드 (`ItemEditSheet.tsx`):** 분수 입력(`1/2`, `1/3` 등)이 가능하여 원재료 재고 파악 시 매우 편리합니다.
    - **진행률 표시:** 카테고리별로 `(완료/전체)` 개수가 표시되어, 실사 누락을 방지하는 효과가 탁월합니다.
    - **임시 저장 (`localStorage`):** 네트워크 불안정이나 앱 재시작 시에도 `localStorage`를 통해 작업 내용이 복원되어 데이터 유실 위험을 줄였습니다.
- **개선 제안:**
    - **조사 초기화 기능:** 실사 도중 실수가 많아 전체를 취소하고 싶을 때, 현재는 개별 수정하거나 브라우저 캐시를 날려야 합니다. '임시저장 초기화' 버튼이 있으면 안전합니다.
    - **검색 바 위치:** 화면 최상단 검색 바는 대화면 스마트폰에서 한 손 조작 시 다소 멀게 느껴질 수 있습니다.

## 2. 관리자 기능 (품목 추가/숨김/복구)의 안전성
품목의 생성 및 관리가 직관적이지만, 유지보수 측면에서 몇 가지 제약이 보입니다.

- **검토 내용:**
    - **하드코딩된 카테고리:** `lib/inventory.ts`와 `InventoryDashboard.tsx`에 카테고리 목록이 하드코딩되어 있습니다. 새로운 카테고리 추가 시 코드 수정이 수반되어야 하는 점은 운영 효율성을 떨어뜨립니다.
    - **품목 숨김/복구:** 삭제 대신 `hidden` 처리를 하여 과거 이력을 보존하는 방식은 매우 안전하고 권장되는 방식입니다 (`lib/inventory-store.ts`).
- **P0 제안:** `current.json`에 존재하는 실제 카테고리 목록을 기반으로 UI를 동적으로 생성하도록 변경이 필요합니다.

## 3. 데이터 손실 방지 및 API 안정성
데이터의 원자적 저장(Atomic Write)과 백업 전략이 매우 훌륭합니다.

- **안전 장치:**
    - **Conflict 감지:** `initialUpdatedAt`을 비교하여 동시 수정 시 덮어쓰기를 방지하고 있습니다 (409 Conflict).
    - **자동 백업:** 모든 저장 및 관리자 작업 직전에 `backups/` 폴더에 `current.json`의 스냅샷을 생성합니다.
    - **이력 기록:** JSON 이력과 더불어 사람이 읽기 쉬운 Markdown 형식(`YYYY-MM-DD.md`)으로 변경 로그를 남깁니다.
- **위험 요소:**
    - **백업 저장소:** 백업이 서버의 로컬 디렉토리에만 저장됩니다. 서버 자체의 디스크 장애에 대비하여 클라우드나 외부 저장소로의 2차 백업 구성 여부를 확인해야 합니다.

## 4. UI/UX 및 완성도 검토
단순한 AI 데모 수준을 넘어 실제 도구로서의 완성도가 높습니다.

- **시각적 요소:**
    - **다크 모드 & 테마:** 현대적인 다크 테마와 적절한 애니메이션(`framer-motion` 유사 스타일)이 적용되어 전문적인 앱 느낌을 줍니다.
    - **이모지 활용:** 품목명에 따른 지능적인 이모지 매칭(`InventoryCard.tsx`의 `getEmoji`)으로 가독성이 좋습니다.
- **속도감:**
    - `next/server`와 클라이언트 상태 관리(`drafts`)가 적절히 분리되어 있어 반응 속도가 매우 빠릅니다.

## 5. 우선순위별 권장 개선 사항 (Next Steps)

### **P0: 데이터 안전 및 무결성 (즉시 확인 필요)**
1. **백업 동기화 확인:** `obsidian-notes/Inventory` 폴더가 외부 클라우드(예: Dropbox, Google Drive, Git 등)로 실시간 백업되고 있는지 확인하십시오.
2. **임시저장 전체 초기화:** 실수로 잘못된 데이터를 많이 입력했을 때를 대비한 '세션 리셋' 기능을 추가하십시오.

### **P1: 운영 편의성 개선 (차주 권장)**
1. **카테고리 동적화:** 하드코딩된 `CATEGORIES` 상수를 제거하고, 데이터 로드 시 존재하는 카테고리를 추출하여 표시하도록 수정하십시오.
2. **최근 변경 내역 뷰어:** 앱 내에서 오늘 누가 어떤 품목을 수정했는지 간단히 리스트로 보여주는 기능을 추가하면 협업 시 혼선을 줄일 수 있습니다.

### **P2: UI/UX 디테일 (여유 시 진행)**
1. **긴 품목명 대응:** 품목명이 아주 길 경우 카드의 레이아웃이 깨지지 않도록 `truncate` 처리와 툴팁을 재점검하십시오.
2. **엑셀 생성기 환경 체크:** 서버의 Python 환경(`inventory-xlsx` venv)이 변경될 경우 엑셀 내보내기가 실패할 수 있으니 에러 핸들링을 강화하십시오.

**종합 의견:** 현재 앱은 소규모 매장의 재고 관리를 위해 충분히 견고하고 실용적으로 설계되었습니다. 위의 P0/P1 개선 사항만 반영된다면 더욱 신뢰할 수 있는 시스템이 될 것입니다.
